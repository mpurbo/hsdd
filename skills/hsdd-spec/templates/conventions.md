# Project Conventions

> Single source of truth for HSDD naming, layout, and established contracts.
> Seeded by hsdd-spec at the root; refined by hsdd-phase-plan per leaf-parent.
> The frontmatter above is machine-read by the `hsdd` CLI; override any value
> and every command honors it.

## Tooling
Install the deterministic CLI once: `npm i -D hsdd` (or use `npx hsdd`).
- `npx hsdd registry`      regenerate contracts/INDEX.md and adr/INDEX.md
- `npx hsdd context X --write`  splice phase X's context into openspec/config.yaml
- `npx hsdd lint`          referential integrity across the tree (CI/pre-commit)
- `npx hsdd status`        derived phase lifecycle (--write emits docs/STATUS.md)
- `npx hsdd check-scope X` diff vs phase X's Touches globs
- `npx hsdd rename A B`    tree surgery (ledger: docs/renames.md)

## Layout (default)
- `docs/spec/{node-id}.md`                    node specs and leaf-parent phase plans
- `docs/verify/{phase-id}.verification.md`    per-phase verification docs
- `docs/STATUS.md`                            generated by hsdd status --write
- `docs/renames.md`                           tree-surgery ledger
- `contracts/{slug}.md` + `contracts/INDEX.md`  first-class contracts (registry generated)
- `adr/{nnn}-{title}.md` + `adr/INDEX.md`      cross-cutting decisions (authored by hsdd-adr, registry generated)
- `schemas/` and `fixtures/`                  contract validation artifacts (stable contracts require one)
- `openspec/config.yaml` + `openspec/changes/` config and one change per phase

## OpenSpec init
Run `openspec init` once, at the repo root (the directory holding this file's
`docs/`, `contracts/`, and `adr/`). One HSDD tree has one OpenSpec project.
Polyrepo: init per repo root and share `contracts/` + `adr/`.

## Naming
- Node id: dotted slug path from root (`acme.backend.auth`)
- Phase id: `{leaf-parent}.{n}` (`acme.backend.auth.3`)
- Contract: `{slug}@v{n}` (`auth-token@v1`)
- ADR: `ADR-{nnn}`; node-local decision: `D{n}`
- User story / acceptance: `US-{n}` / `AC-{n}.{y}`
- Phase branch / worktree: `hsdd/{phase-id}` (one worktree per active phase when
  phases run concurrently; required, not suggested)

## Phase design
- Ordering: per the `ordering_policy` frontmatter above (default interfaces-first)
- <= 8 OpenSpec tasks per phase; each phase is one PE: the largest change one
  reviewer can genuinely review and manually verify in a single sitting
  (~<= 400 changed lines of non-generated code, ~half a working day end to end)
- Review tiers: gate-only | spot-check | full-review; a phase whose outputs feed
  2+ later phases is spot-check minimum
- A phase producing a contract runs its contract verification in the gate;
  consumers test against the fixtures, never hand-rolled mocks
